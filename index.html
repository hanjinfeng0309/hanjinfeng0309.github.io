ArchLinux搭建微信网页开发环境

Posted on 2020-07-27 Edited on 2020-07-30

微信开发需要开发者拥有自己的服务器资源(必须是80或443端口)来进行配置，但是很多初学者或者大部分人干脆就没有自己的服务器，这时可以采用内网穿透的方式，将本地的80端口开放出来，能让外界访问到。

这里推荐natapp，优点是有免费隧道可以使用，缺点是每次启动都是不同的域名，得随时更换自己的配置情况。

由于Linux默认不让普通用户使用小于1024一下的端口，而微信公众号开发配置中接口的域名又必须是80端口，所以可以把应用放到8080(或是其它能用的)端口，用防火墙iptables进行端口转发，将80端口的内容转发到8080端口来。

本次将采用Springboot作为微信网页的后台服务，前端页面使用vue搭建，并记录我在此过程中所遇到的一些坑。

一、开启端口转发

1、首先得机器上安装有iptables：sudo pacman -S iptables，若已安装，则无视这一步。

2、启动iptables服务：

3、开启端口转发： sudo vim /etc/sysctl.conf，输入net.ipv4.ip_forward=1

4、启用/etc/sysctl.conf：sudo sysctl -p

二、将80端口转发到8080端口

sudo iptables -t nat -A OUTPUT -p tcp -d 127.0.0.1 --dport 80 -j DNAT --to 127.0.0.1:8080

使用sudo iptables -t nat -L --line-number可以查看详情

七、通过网页授权获取用户信息

根据微信开发文档，获取用户信息分为以下步骤：

  • 1）引导用户进入授权页面同意授权，获取code
  • 2）通过code换取网页授权access_token（与基础支持中的access_token不同）
  • 3）如果需要，开发者可以刷新网页授权access_token，避免过期
  • 4）通过网页授权access_token和openid获取用户基本信息（支持UnionID机制）

根据描述，我们可以拼接出我们的链接来获取code，并通过code获取用户信息了。

访问链接：https://open.weixin.qq.com/connect/oauth2/authorize?appid=wx0134988a6c1ee22a&redirect_uri=http%3A%2F%2Fsjyqd8.natappfree.cc%2Findex&response_type=code&scope=snsapi_base&state=openId#wechat_redirect

其中redirect_uri参数要填写自己要跳转的页面，该页面处于配置网页授权时填写的域名下(我这里由于重启过一次natapp，导致我的域名发生了变化，和之前不一样了，所以要重新配置)。在当前页面跳转成功后，页面地址会变成redirect_url参数中的地址加上?code=CODE&state=STATE参数，比如此处就是http://sjyqd8.natappfree.cc/index?code=xxx&state=openId，接下来我们code参数取出来，放到我们的页面展示。

Kafka消费指定offset的消息

Posted on 2020-06-22

假定有这样一个场景：Kafka的消费者从生产者接收消息，并对每一条消息进行处理，如果处理失败，则认为Kafka消费该条消息失败，需要重新消费该条消息，一直到成功为止。

思路：

对每条消息记录其偏移量offset，消费失败后，记录下offset然后重新提交该offst，并从该offset进行重新消费。
