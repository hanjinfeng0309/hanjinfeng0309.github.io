ArchLinux搭建微信网页开发环境

Posted on 2020-07-27

微信开发需要开发者拥有自己的服务器资源(必须是80或443端口)来进行配置，但是很多初学者或者大部分人干脆就没有自己的服务器，这时可以采用内网穿透的方式，将本地的80端口开放出来，能让外界访问到。
这里推荐natapp，优点是有免费隧道可以使用，缺点是每次启动都是不同的域名，得随时更换自己的配置情况。

由于Linux默认不让普通用户使用小于1024一下的端口，而微信公众号开发配置中接口的域名又必须是80端口，所以可以把应用放到8080(或是其它能用的)端口，用防火墙iptables进行端口转发，将80端口的内容转发到8080端口来。

本次将采用Springboot作为微信网页的后台服务，前端页面使用vue搭建，并记录我在此过程中所遇到的一些坑。

一、开启端口转发

1、首先得机器上安装有iptables：sudo pacman -S iptables，若已安装，则无视这一步。

2、启动iptables服务：

3、开启端口转发： sudo vim /etc/sysctl.conf，输入net.ipv4.ip_forward=1

4、启用/etc/sysctl.conf：sudo sysctl -p

二、将80端口转发到8080端口

sudo iptables -t nat -A OUTPUT -p tcp -d 127.0.0.1 --dport 80 -j DNAT --to 127.0.0.1:8080

使用sudo iptables -t nat -L --line-number可以查看详情

五、申请微信公众号测试号并进行配置

申请微信公众号测试号异常简单，进入微信公众帐号测试号申请系统，然后使用微信扫码就可以拥有属于自己的测试账号了。

在填写接口配置信息并提交的时候，先要在后台搭建好相应的接口。

修改之前的IndexController.java

根据微信接口接入指南，需要对消息进行校验，步骤如下：

1）将token、timestamp、nonce三个参数进行字典序排序

2）将三个参数字符串拼接成一个字符串进行sha1加密

3）开发者获得加密后的字符串可与signature对比，标识该请求来源于微信
